## 1. 任务目标

完成 v1.2 全量验收：运行类型检查、构建、单元/集成测试，验证需求覆盖，生成验收报告，并将设计文档、执行计划、阶段 Spec、任务 Spec 归档。

## 2. 对应需求与验收标准

| 需求编号 | 需求描述 | 本任务验收标准 |
|---|---|---|
| REQ-1.2-1 | code-quality 源码迁移到 legacy-shield 内部 | 验收报告包含源码迁移验证 |
| REQ-1.2-2 | 保留全部现有能力 | 验收报告包含 `all` / `module` / `diff` 能力验证 |
| REQ-1.2-3 | `quality` 子命令 CLI 完全兼容 | 验收报告包含 CLI 兼容验证 |
| REQ-1.2-4 | 不修改老项目源码，不在老项目内安装依赖 | 验收报告包含零侵入验证 |
| REQ-1.2-5 | 用户文档更新 | 验收报告包含文档更新验证 |

## 3. 实现步骤

### 3.1 工程检查

- 运行 `pnpm typecheck`；
- 运行 `pnpm build`；
- 运行 `pnpm test`；
- 确认全部通过。

### 3.2 需求覆盖验证

- 对照 `requirements-v1.2.md` 逐项确认实现；
- 记录每项需求的验证方式与结果。

### 3.3 端到端验证

- 运行 `node ./dist/cli.js quality --project <legacy>`；
- 验证 `all`、`module`、`diff` 三种命令的输出与退出码；
- 验证配置 `CODE_QUALITY_ROOT` 时打印废弃提示且运行正常。

### 3.4 生成验收报告

- 创建 `docs/specs/acceptance-report-v1.2.md`；
- 包含：需求覆盖、测试结果、遗留问题、签字确认。

### 3.5 文档归档

- 将 `design-v1.2.md`、`execution-plan-v1.2.md`、`phase-v1.2-spec.md` 状态更新为「已完成，已归档」；
- 将所有任务 Spec 状态更新为「已完成，已归档」；
- 确保不再修改已归档文档。

## 4. 测试计划

### 4.1 单元测试

- 运行 `pnpm test`，确认单元测试通过。

### 4.2 集成测试 / 端到端测试

- 运行端到端验证脚本，确认 `quality` 子命令行为正确。

### 4.3 回归测试

- 确认 v1.1 核心功能（除 code-quality 路径变化外）未受影响；
- 确认 legacy-shield 自身 `pnpm test` 通过。
