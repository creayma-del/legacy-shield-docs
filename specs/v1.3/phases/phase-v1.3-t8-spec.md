## 3. 实现步骤

### 3.1 更新 `README.md`

- 说明 v1.3 新增定位：业务系统开发阶段护航。
- 说明监控对象扩展为 Web 端与移动端 H5。
- 新增 `quality` 子命令参数说明表（平台、内存/资源监控、日志相关）。
- 补充快速开始示例：H5 项目监控、内存监控、资源监控。

### 3.2 更新 `docs/usage.md`

- 补充 H5/Web 监控示例。
- 补充内存泄漏监控示例（`--enable-memory-monitor`）。
- 补充资源加载长耗时监控示例（`--enable-resource-monitor`）。
- 说明结构化日志路径 `<project>/.legacy-shield/logs/<sessionId>.ndjson` 与字段含义。
- 说明如何显式覆盖平台类型、如何禁用特定规则。

### 3.3 更新 `docs/api.md`

- 若受影响，补充 `QualityCommandOptions`、`StructuredLogEntry`、`MemoryMonitorResult`、`ResourceMonitorResult` 等类型说明。
- 补充 `lib/platform.ts`、`lib/runtime-monitor/index.ts` 的公共接口说明。

### 3.4 不修改已归档文档

- 不修改已归档的 Phase 1-5 及 v1.1、v1.2 Spec 文档。
- 不修改 `docs/specs/acceptance-report-v1.2.md` 等已归档验收报告。

### 3.5 全量验证

- 运行 `pnpm typecheck`。
- 运行 `pnpm build`。
- 运行 `pnpm test`。
- 使用示例项目验证 `quality` 子命令新增参数端到端行为。

### 3.6 生成验收报告

- 创建 `docs/specs/acceptance-report-v1.3.md`。
- 包含：测试环境、测试范围、通过/不通过项、遗留问题、评审结论。

### 3.7 专家评审

- 调用测试验收专家对 T1-T7 实现与 T8 文档进行评审。
- 根据评审意见修订后，按以下顺序归档：
  1. 将 T1-T8 任务 Spec 状态更新为「已完成，已归档」；
  2. 将阶段 Spec 状态更新为「已完成，已归档」；
  3. 同步归档设计文档与执行计划（更新状态为「已完成，已归档」）。

## 4. 测试计划

### 4.1 单元测试

- 确认 T1-T7 的所有单元测试均通过。

### 4.2 集成测试 / 端到端测试
